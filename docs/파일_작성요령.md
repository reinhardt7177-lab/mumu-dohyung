## 3. 비주얼 표준 (지브리 픽셀 월드)

### 캔버스 사양
- **로지컬 사이즈**: `W=384, H=216` (16:9)
- **확대 배율**: `PX=4` → 실제 1536×864 픽셀 출력
- `imageSmoothingEnabled = false`, CSS `image-rendering: pixelated`
- **letterbox**: `width: min(100vw, calc(100vh * 16/9))`, `height: min(100vh, calc(100vw * 9/16))`

### 컬러 팔레트
| 용도 | HEX | 비고 |
|---|---|---|
| 그림자 | `#1a1428` | **검정 절대 금지** — 보랏빛 그림자 사용 |
| 배경 베이스 | 차시별 (1차시: 따뜻한 앰버/올리브) | 차시 톤 결정 자유 |
| 골드 강조 | `#f5d878` | 타이틀·HUD 강조 |
| 본문 | `#f5e6c8` | 카드 안 텍스트 |
| 보조 | `#c8b890` | 라벨·설명 |
| 학습 강조(초록) | `#a8c890` | "🌱 발견한 사실" 표기 |
| 도형 4종 | square=황토 / triangle=적 / circle=청 / pentagon=보라 | (1차시 표준) |

### 타일 한 칸
- **TILE = 16 logical px** (= 실제 64 px)
- 잔디(짝)/흙(홀) 체크무늬 배경에 골 자리는 돌 타일.
- 격자선은 `rgba(20,12,8,0.85)` (셀 경계 명확화)

### 노이즈/효과
- 배경에 `noise(ctx, 0, 0, W, H, 0.025)` 한 줄 (수채화 톤)
- 비네팅: 화면 중심→가장자리 보라색 그라데이션 0.35
- 캐릭터/도형 그림자: `shadow(ctx, cx, cy, r, alpha)` 사용

## 4. UI 표준

### 라우팅
1차시처럼 `App.tsx`에서 `view` 상태로 라우팅:
```ts
type View = 'intro' | 'select' | 'play' | 'finale';
```
- **intro**: 차시 부제 + 임무 보고 카드 + ▶ 시작하기
- **select**: 5×4 (또는 5×N) 스테이지 그리드, 티어별 색상, 클리어 시 🏺
- **play**: `<Scene stageIndex onClear onExit />`
- **finale**: 누적 통계 + "다음 차시: ..." 안내

### 클리어 추적
```ts
interface ClearedRecord {
  /* 차시별 메트릭 (1차시: moves, pushes / 2차시: moves, flips ...) */
}
const [cleared, setCleared] = useState<Record<number, ClearedRecord>>({});
```
- 다음 스테이지 자동 진행 (Enter 또는 모바일 탭)
- 마지막 스테이지 클리어 시 finale로 전환

### 스테이지 선택
- 티어별 색상 (5티어 표준): A=`#a8c890` / B=`#c8c478` / C=`#c8a878` / D=`#c87878` / E=`#a878c8`
- 직전 스테이지 미클리어 시 🔒 잠금
- 클리어 시 🏺 + 메트릭(예: `✓ 12이동/4민`) 표시

### HUD (게임 화면 상·하단 패널)
- 상단: `[차시 아이콘] N차시 | 스테이지명` (좌) + `메트릭 | 키 안내` (우)
- 하단: `💡 hint` (스테이지 데이터의 hint)
- 클리어 배너: 중앙 모달 + `🌱 lesson` (학습 포인트) + Enter 깜빡이

## 5. 코드 패턴

### `pixel.ts` (재사용 헬퍼 — 변경 금지)
```ts
export const PX = 4;
export const W = 384;
export const H = 216;
export const px(...);  rect(...);  dither(...);  noise(...);
export const glow(...); shadow(...);  sprite(map, colors, ox, oy);
```
1차시에서 이미 검증된 헬퍼 그대로 복사. 차시별로 추가 헬퍼가 필요하면 `pixel.ts` 하단에 추가 (기존 함수 시그니처 변경 금지).

### `logic.ts` (차시 고유)
- `GameState` 인터페이스 (cols, rows, walls, player, ... + anim)
- `loadStage(stage)`: `Stage`를 받아 초기 상태 생성
- `try<Action>(state, ...)`: 학습 동작별 상태 전이 함수 (1차시 `tryMove`, 2차시 `tryMove` + `tryFlip` ...)
- `tickAnim(state, dt)`: 애니메이션 진행 (0.14s 표준)
- `checkCleared(state)`: 모든 골 일치 여부 → cleared 갱신

### `stages.ts` (차시 고유)
- 20 스테이지 / 5티어 (4×5) 표준
- 각 스테이지: `{ id, tier, name, hint, lesson, grid, ...메타 }`
- **lesson** 필드는 클리어 배너에 표시되는 핵심 학습 메시지 (한 문장)
- grid는 텍스트 배열로 가독성 우선 (`. # P S T O N` 등)

### `Scene.tsx` (차시 고유 렌더 + 입력)
- `useRef`로 `GameState`, `tickRef`, `particlesRef` 관리 (불필요 리렌더 회피)
- `requestAnimationFrame` 루프에서 `tickAnim` + `draw`
- 입력은 터치/클릭 전용 (`TouchControls`의 버튼 또는 캔버스 click) — 키보드 안내·표기 노출 X
  · 내부 dispatch는 `KeyboardEvent`를 `window.dispatchEvent` 하지만 사용자에게 키보드 사용을 안내하지 않음
- 클리어 시 입자 폭발 + 배너 표시 → Enter로 다음 스테이지

## 6. 모바일/태블릿 (TouchControls)

1차시 표준:
- 좌측 하단: D-pad (4방향)
- 우측 하단: 되돌리기(↩) / 다시 / 메뉴 액션
- 햅틱: `navigator.vibrate(15)`
- 표시 조건: `'ontouchstart' in window || maxTouchPoints > 0 || matchMedia('(pointer: coarse)') || innerWidth < 1100`

차시별 추가 액션이 있으면(예: 2차시 ↔/↕ 뒤집기) **우측 액션 컬럼에 추가**, D-pad는 그대로 둘 것.

## 8. 학습 요소 일관성

각 차시 게임은 다음 3가지를 **반드시** 표면화한다:

1. **인트로 카드**: 학습 목표(녹색 강조)를 명시
2. **스테이지별 hint**: 1줄 조작 힌트
3. **스테이지별 lesson**: 클리어 시 학습 포인트 (한 문장, 핵심 개념 강조)
4. **피날레**: "발견한 사실" — 차시 핵심 개념을 한 문단으로 정리, 다음 차시 안내

> 게임은 도구일 뿐, **학습 메시지가 화면에 항상 보이도록** 한다.
